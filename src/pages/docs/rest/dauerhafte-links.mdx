# Statische Daten

Wir gleichen unsere Daten regelmäßig mit denen der statistischen Ämter ab und halten damit unsere Datenbasis auf dem neuesten Stand. Das heißt aber auch, dass die gleichen Abfragen für die REST-Schnittstelle zu unterschiedlichen Zeitpunkten unterschiedliche Ergebnisse liefern können, zum Beispiel wenn neue Daten hinzugefügt oder Datensätze aktualisiert werden.

Wir stellen deshalb mit https://static.tabular.genesapi.org eine statische API zur Verfügung, die genauso funktioniert wie https://tabular.genesapi.org – nur, dass die hier generierten Tabellen nicht aktualisiert werden und deshalb immer genau die Daten zurückgeben, die du ursprünglich abgerufen hast.

## Abfragen über die „normale“ REST-Schnittstelle

Das Folgende Beispiel liefert Zahlen zur Einbürgerung von Ausländern mit einer Aufenthaltsdauer von unter 8 Jahren in allen brandenburgischen Landkreisen, im Zeitraum seit 2000:

```js
https://tabular.genesapi.org/?region=all&level=3&parent=12&time=2000:&data=12511:BEV008(AUFDA1:AUFDA00B08)
```

## Statische Abfrage-Links erstellen

Damit diese Abfrage dauerhaft und verlässlich das selbe Ergebnisse liefert, kannst du die Subdomain `static` verwenden. Alle Parameter bleiben dabei gleich.

```js
https://static.tabular.genesapi.org/?region=all&level=3&parent=12&time=2000:&data=12511:BEV008(AUFDA1:AUFDA00B08)
```

Wenn du diese URL aufrufst wird eine statische Version der Abfrage generiert und abgelegt. Du wirst an eine neue URL weitergeleitet, die dann dauerhaft genau das Ergebnis zurückliefert, das du zu diesem Zeitpunkt abgerufen hast.

Eine Schritt-für-Schritt-Anleitung wie die URL für die Abfrage der Schnittstelle aufgebaut sein muss, findest du unter [Aufbau einer REST-Abfrage](abfragen).
